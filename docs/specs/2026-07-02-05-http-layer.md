## Objetivo

Expor os use-cases via REST API com validação de entrada (Zod), serialização de saída (Presenter), documentação automática (Swagger) e tratamento global de erros.

---

## Rotas

| Método | Rota | Use-Case | Descrição |
|--------|------|----------|-----------|
| `POST` | `/projects` | `CreateProjectUseCase` | Criar projeto |
| `GET` | `/projects` | `ListProjectsUseCase` | Listar todos |
| `GET` | `/projects/:id` | `GetProjectUseCase` | Buscar por ID |
| `PATCH` | `/projects/:id` | `UpdateProjectUseCase` | Atualizar campos |
| `DELETE` | `/projects/:id` | `DeleteProjectUseCase` | Deletar |
| `PATCH` | `/projects/:id/status` | `UpdateProjectStatusUseCase` | Avançar status |
| `GET` | `/projects/:id/ai-analysis` | `GetAiAnalysisUseCase` | Análise de IA |

## DTOs (Validação com Zod)

Os DTOs usam **nomes em português** (conforme a API pública). O `ZodValidationPipe` aplica o schema Zod ao body antes de chegar no método do controller.

### `CreateProjectDto`
```typescript
{
  nome: string (1–200 chars)
  dataInicio: Date (coerced)
  previsaoTermino: Date (coerced, deve ser > dataInicio)
  orcamentoTotal: number (positivo)
  descricao: string (não vazia)
}
```

### `UpdateProjectDto`
Todos os campos opcionais (Partial do CreateProjectDto).

### `UpdateStatusDto`
```typescript
{ status: ProjectStatus }  // enum nativo via z.nativeEnum
```

## Presenter (`ProjectPresenter`)

**Arquivo:** `src/projects/http/project.presenter.ts`

Converte `Project` (domínio, inglês) → `ProjectHttpResponse` (HTTP, português):

```typescript
{
  id, nome, dataInicio, previsaoTermino,
  orcamentoTotal, descricao, status, risco,
  criadoEm, atualizadoEm
}
```

Datas são serializadas como ISO 8601 string.

---

## `ZodValidationPipe`

**Arquivo:** `src/shared/pipes/zod-validation.pipe.ts`

Pipe NestJS que recebe um schema Zod e lança `BadRequestException` (400) com os erros formatados se a validação falhar.

---

## `HttpExceptionFilter`

**Arquivo:** `src/shared/filters/http-exception.filter.ts`

Filter global que captura todas as `HttpException` e retorna JSON padronizado:
```json
{ "statusCode": 404, "message": "Project not found" }
```

Registrado em `AppModule` via `APP_FILTER` para cobrir todos os controllers.
